# PARTE B — Programa C com write() direto

#include <unistd.h>

int main() {
    const char msg[] = "Modo usuario -> kernel via write()\n";
    write(1, msg, sizeof(msg) - 1);
    return 0;
}

# PARTE C — Questões de síntese

# 1. A apresentação afirma que "instruções de I/O são privilegiadas".
#    Como um programa de usuário consegue fazer I/O sem executar
#    instruções privilegiadas diretamente? (mínimo 2 frases)

Resposta 1: O programa do usuário não mexe no hardware diretamente. Ele faz um pedido para o sistema operacional usando uma chamada de sistema (syscall). Quando isso acontece, o processador muda para o modo kernel (modo protegido), e é o próprio sistema operacional que faz o trabalho de I/O em nome do programa de forma segura.


# 2. Compare write(fd, buffer, tamanho) com o "programa de canal"
#    da apresentação: o que os dois têm em comum em termos de
#    responsabilidades?

Resposta 2: Em termos de responsabilidades, tanto a chamada write(fd, buffer, tamanho) quanto um programa de canal têm em comum o papel de isolar e especificar os parâmetros exatos de uma transferência de dados para que o hardware/sistema possa executá-la de forma independente.
